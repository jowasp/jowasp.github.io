Developers new to security

Quick intro guide

ZAP is an Intercepting Proxy. It allows you to see all of the requests you make to a web app and all of the responses you receive from it. Amongst other things this allows you to see AJAX calls that may not otherwise be obvious.With ZAP you can learn how to implement and use ZAP early on in the software development lifecycle. In addition, ZAP can help you learn about the most common security flaws. There are many resources that can help you understand how to achieve this such as

  1. Begin by examining documentation such as the OWASP top 10, SANS Top 25 or the TOP ten security flaws.
  2. Learn how to Implement a Secure Software development lifecycle and use ZAP to support it.
  3. Check SDLC frameworks such as OpenSamm

Explore ZAP

Get you hands dirty hacking

You will need to configure your browser to use ZAP as a proxy. By default ZAP uses an Address of 'localhost' and a Port of '8080', but these can be changed via the Options Local Proxy screen. To learn more on this, please visit the Start Guide

Quick Start

Use the “Quick Start” tab to see ZAP in action.In the textbox "URL to attack" write the URL of the website you want to "attack". Please do not use this in applications deployed into production environment! Check the OWASP Web Testing Environment or the OWASP Broken Web Applications Project for a complete test environmnet

My First SQL Injection with ZAP!

Begin testing your application against one or two of the most common security flaws such as SQL injections. You can wether use the OWASP Web Testing Environment or the OWASP Broken Web Applications Project. There are many dummy vulnerable websites that you can use for this purpose such as WebGoat or Damn Vulnerable Website.
One of the ZAP modules that helps you test this is the Active Scan Rules. This scanner scans for SQL Injection vulnerabilities in an RDBMS-independent fashion.This scanner may be able to fingerprint the RDBMS if the application throws a known RDBMS specific SQL error message. It does not exploit any RDBMS specific techniques, and so is the best SQL injection scanner to use as a starting point.
